# 通讯

Roplat 将通讯分为两类：主流通讯与旁路通讯。

## 主流通讯

主流通讯由节点输入输出关系决定，是系统拓扑的主干数据流。

特点：

1. 强类型
2. 可参与编译期拓扑分析
3. 是系统调度优化的重要依据

## 旁路通讯

旁路通讯用于不适合直接放在节点 I/O 主干上的数据交换。

当前最核心的是三缓冲（SPMC 状态更新）：

1. 写入和读取本身不保证原子
2. 指针交换是原子操作
3. 通过“写缓冲 + ready 槽位交换”实现低开销最新值传播

## 透明类型与不透明类型

### 透明类型

声明方式：

```rust
#[roplat::roplat_msg]
#[repr(C)]
pub struct SensorData {
    pub x: f64,
    pub y: f64,
}
```

特点：

1. 参与跨语言布局协作
2. 自动生成跨语言描述产物
3. 自动实现 `TypeBinding<CppLang>` 与 `TypeBinding<PyLang>`

### 不透明类型

声明方式：

```rust
#[roplat::roplat_msg(lang = "cpp")]
pub struct CppData;
```

当前语义：

1. Rust 侧只保留类型标记
2. 具体字段与行为由目标语言用户文件实现
3. 默认只生成用户文件，不生成系统基类
4. 模板注释保留 alias/wrapper 可选方案

## TypeBinding 的作用

`TypeBinding<L>` 是多语言映射桥梁。

它用于回答两个问题：

1. 目标语言中的类型名是什么
2. 这个类型是否是 opaque

并且 `Publisher<T>`、`Subscriber<T>`、`TripleBufferChannel<T>` 都会把绑定能力转发到 `T`。

## 新人建议

1. 能透明就透明，优先保证类型可见性。
2. opaque 只在确实需要语言私有实现时使用。
3. 先跑通主流通讯，再引入旁路通讯。
