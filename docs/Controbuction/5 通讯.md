# 通讯

我们将通讯分为主流通讯和旁路通讯两部分，主流通讯是节点输入输出所代表的通讯，这类通讯方式损耗最小；旁路通讯则是节点之间的其他通讯方式，如节点内的通讯，这类通讯不受编译的静态输入输出的管理，但是仍然在编译过程中被搭建。这种通讯方式可能会涉及到运行时的空间开辟和数据复制，因此损耗较大。

## 主流通讯

主流通讯通过节点的输入输出所绑定，由 rust 来管理。

主流通讯作为系统编排的依据，将在编译期被用于系统的编排和优化，编译器将根据节点的输入输出类型来推断节点之间的通讯关系，并进行相应的优化，如内存布局优化、数据传输优化等。

在拓扑排序优化后，主流通讯表现为相同线程内的异步节点之间的移动语义或引用传递。

## 旁路通讯

旁路通讯存在很多不同的通讯方式，不同的通讯方式具备不同的通讯内核，出于对多语言通讯的需要，这里的通讯内核主要被涉及为 C 语言的 FFI 接口，被不同的语言封装为不同的高级特征。

### 三缓冲内存 —— 状态更新

这类通讯方式始终保持最新的数据，其功能类似 `ROS` 中的话题

三缓冲的核心在于内存的写入和读取不是原子的，而指针的交换是原子的，因此在写入数据时，先将数据写入一个缓冲区，然后将指针交换到另一个缓冲区，这样读取数据时就可以保证读取到的是完整的数据，而不会出现读取到半个数据的情况。

空间的开辟是懒惰的，因为消息不一定具备默认构造函数，因此只有在写入的时候会开辟空间。在此过程中，空指针将被封装为不同语言中的高级数据结构用于表述数据的有效性，如 Rust 中的 `Option`，C++ 中的 `Optional`，Python 中的 `None` 等。

具体的来说，分为以下几个步骤：

#### 编写内核

编写支持原子指针交换的 rust 低层三缓冲内核，并提供 C 语言的 FFI 接口。（说是三缓冲，但是由于 spmc 的特性，实际上需要 N+2 个缓冲区，其中一个 `Writer` , 一个 `Ready` , 和 N 个 `Reader` ）

```rust
// roplat_core/src/comm/triple_buffer.rs
pub struct TripleBufferCtrl {
    ready: AtomicPtr<c_void>,
}

#[no_mangle]
pub unsafe extern "C" fn roplat_tb_swap(ctrl: *mut TripleBufferCtrl, local: *mut c_void) -> *mut c_void {
    (*ctrl).ready.swap(local, Ordering::AcqRel)
}
```

#### 不同语言的高级封装

##### Rust

```rust
impl<T> Subscriber<T> {
    pub fn get_latest(&mut self) -> Option<&T> {
        let next = unsafe { roplat_tb_swap(self.ctrl, self.read_buf as *mut c_void) };
        self.read_buf = next as *mut T;
        
        if self.read_buf.is_null() {
            None
        } else {
            unsafe { Some(&*self.read_buf) }
        }
    }
}

pub struct Publisher<T> {
    ctrls: Vec<*mut TripleBufferCtrl>,
    write_buf: *mut T, // 初始为 null
}

impl<T> Publisher<T> {
    pub fn publish(&mut self, data: T) {
        // 1. 如果写缓冲区为空，直接接管数据的所有权并移动到堆上
        if self.write_buf.is_null() {
            self.write_buf = Box::into_raw(Box::new(data));
        } else {
            // 2. 如果已有缓冲区，将新数据“写”进去
            unsafe { std::ptr::write(self.write_buf, data); }
        }

        // 3. 广播交换
        unsafe {
            for &ctrl in &self.ctrls {
                let next = roplat_tb_swap(ctrl, self.write_buf as *mut c_void);
                if next.is_null() {
                    // 惰性扩张：如果换回的是空，说明系统缺块内存，我们需要再造一个
                    // 注意：这仅发生在冷启动的前几次调用
                    self.write_buf = Box::into_raw(Box::new(std::ptr::read(self.write_buf)));
                } else {
                    self.write_buf = next as *mut T;
                }
            }
        }
    }
}
```

##### C++

C++ 中需要自己实现一个 `Optional` 类型来管理数据的有效性

```cpp
// roplat_cpp/include/roplat/comm.hpp
template <typename T>
class Subscriber {
    void* ctrl;
    T* read_buf = nullptr; // 初始为空

public:
    // 返回一个类似 Option 的轻量视图
    class Option {
        T* ptr;
    public:
        Option(T* p) : ptr(p) {}
        bool is_some() const { return ptr != nullptr; }
        const T& unwrap() const { return *ptr; }
        const T* operator->() const { return ptr; }
    };

    Option get_latest() {
        // 尝试交换。如果 Publisher 还没发过数据，换回来的依然是 nullptr
        void* next = roplat_tb_swap(ctrl, (void*)read_buf);
        read_buf = (T*)next;
        return Option(read_buf);
    }
};

template <typename T>
class Publisher {
    std::vector<void*> ctrls;
    T* write_buf = nullptr;

public:
    void publish(const T& data) {
        if (!write_buf) {
            // 第一次发布时，如果没有写缓冲区，则进行开辟（调用非默认构造函数）
            write_buf = new T(data); 
        } else {
            *write_buf = data;
        }

        // 广播交换
        for (void* c : ctrls) {
            void* next = roplat_tb_swap(c, (void*)write_buf);
            if (!next) {
                // 如果 Ready 槽位是空的，说明我们要为这个订阅关系新“生产”一个缓冲区
                // 这样通过几次 publish，系统会自动收敛到 3 缓冲状态
                write_buf = new T(data);
            } else {
                write_buf = (T*)next;
            }
        }
    }
};
```

##### Python

```python
# roplat_py/comm.py
class Subscriber:
    def get_latest(self) -> Optional[T]:
        self._ptr = lib.roplat_tb_swap(self._ctrl, self._ptr)
        if not self._ptr:
            return None
        return self._ctype.from_address(self._ptr)

class Publisher(Generic[T]):
    def __init__(self, ctrls: List[int], ctype_struct: Type[T]):
        self._ctrls = ctrls
        self._ptr = 0 # 初始为空指针
        self._ctype = ctype_struct

    def publish(self, data: T):
        if self._ptr == 0:
            # 第一次发布：创建内存并拷贝数据
            obj = self._ctype()
            # 假设 data 是一个包含属性的对象
            pointer_copy(obj, data) 
            self._ptr = ctypes.addressof(obj)
            # 保持引用防止被 Python GC 掉
            _KEEP_ALIVE[self._ptr] = obj
        else:
            # 覆写现有内存
            target = self._ctype.from_address(self._ptr)
            pointer_copy(target, data)

        for ctrl in self._ctrls:
            next_ptr = lib.roplat_tb_swap(ctrl, self._ptr)
            if next_ptr == 0:
                # 惰性扩张
                new_obj = self._ctype()
                pointer_copy(new_obj, data)
                self._ptr = ctypes.addressof(new_obj)
                _KEEP_ALIVE[self._ptr] = new_obj
            else:
                self._ptr = next_ptr
```

### 环形队列 —— 事件通知

这类通讯方式努力保管所有的塞入数据，并随着需求被逐个取出，在不超过环容量的情况下，不会丢失数据

## 通讯数据

通讯的数据是重要的，尤其是在多语言中。通讯数据可以被大体分为两类，一类是语言间通讯的数据，通常被绑定为 C 结构体，通常被成为透明类型；另外一类是单一语言内通讯的数据，对于其他语言来说也被称为不透明数据。但是无论哪种数据，都需要在 rust 中注册一个对应的数据结构用于调度。

### 透明类型

透明类型使用 `#[roplat::roplat_msg]` + `#[repr(C)]` 声明。

特性：

- 自动生成跨语言描述信息（用于构建期落盘）
- 自动实现 `TypeBinding<CppLang>` 和 `TypeBinding<PyLang>`
- 可作为节点端口消息类型、也可被三缓冲 pub/sub 直接承载

当前生成规则：每个透明类型生成一个系统管理文件（默认 C 头文件）。

### 不透明数据

不透明数据使用 `#[roplat::roplat_msg(lang = "cpp"|"py")]` 声明。

特性：

- 仅为目标语言生成 `TypeBinding`（另一语言不实现）
- `is_opaque() == true`
- 构建期只生成一个用户文件（按 `lang` 选择目录）

不透明数据不会按透明消息方式跨语言共享布局；其本质是“Rust 调度侧可见、目标语言实现侧可用”的单语言数据镜像。

### 类型绑定与通讯泛型

为统一节点字段、端口和三缓冲类型映射，roplat 引入了：

- `roplat::type_binding::TypeBinding<L>`
- 语言标记：`CppLang`、`PyLang`

三缓冲类型实现了绑定转发：

- `Publisher<T>: TypeBinding<L>`（当 `T: TypeBinding<L>`）
- `Subscriber<T>: TypeBinding<L>`（当 `T: TypeBinding<L>`）
- `TripleBufferChannel<T>: TypeBinding<L>`（当 `T: TypeBinding<L>`）

因此在多语言节点中使用三缓冲相关类型时，映射可以由类型系统自动推导并在编译期校验。
