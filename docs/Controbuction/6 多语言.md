# 6 多语言

## 多语言节点

除了 Rust 外，多语言节点需要在 Rust 侧声明一个“傀儡节点”（调度镜像）。

实现顺序：

1. 在 Rust 中写节点镜像结构和 input/output 描述。
2. 通过过程宏提取元数据。
3. 由 `build.rs` 两遍扫描生成目标语言文件。

每个多语言节点会生成两类文件：

- 系统管理基类：每次构建覆盖，用于保持与 Rust 声明同步。
- 用户实现文件：仅首次生成，后续不覆盖，允许用户写业务逻辑。

当前默认目录：

- 基类：`cpp/roplat_gen`、`py/roplat_gen`
- 用户文件：`cpp/src`、`py`

以上目录可在 `build.rs` 中通过 `BuildOrchestrator` 自定义。

### 节点宏写法

```rust
#[roplat::node(
    lang = "cpp",
    input(sensor_input, SensorData),
    output(motor_output, MotorCommand),
)]
pub struct CppController {
    gain: f64,
    offset: f64,
}
```

说明：

- `lang` 必填，支持 `cpp` 或 `py`
- 必须同时给出至少一个 `input(...)` 和一个 `output(...)`
- 节点名建议直接与目标语言类名一致，不加额外后缀

## 多语言数据

见 [](./5%20%E9%80%9A%E8%AE%AF.md)

多语言数据统一使用 `#[roplat::roplat_msg]`：

1. 透明数据（跨语言可见）

```rust
#[roplat::roplat_msg]
#[repr(C)]
pub struct SensorData {
    pub x: f64,
    pub y: f64,
}
```

- 默认作为透明消息处理
- 每个类型生成一个系统管理文件（当前默认是 C 头文件）

2. 不透明数据（单语言可见）

```rust
#[roplat::roplat_msg(lang = "cpp")]
pub struct CppData {
    scale: f64,
    mode: i32,
}
```

- `lang` 指定目标语言
- 每个类型仅生成一个用户文件（不生成系统基类）

在傀儡节点中的类型映射采用统一特征：`roplat::type_binding::TypeBinding<L>`。

- 语言标记：`CppLang`、`PyLang`
- 核心接口：
    - `target_type_name() -> &'static str`：目标语言类型名
    - `is_opaque() -> bool`：是否为不透明类型

绑定实现策略：

1. 基础类型内建实现

- `f32/f64/i32/u64/bool/...` 均实现 `TypeBinding<CppLang>` 与 `TypeBinding<PyLang>`。

2. 三缓冲通讯类型自动绑定

- `Publisher<T>`、`Subscriber<T>`、`TripleBufferChannel<T>` 实现 `TypeBinding<L>`，并转发到 `T: TypeBinding<L>`。
- 这样在节点端口中使用 pub/sub 时，类型映射可自动推导。

3. 透明消息自动绑定

- `#[roplat::roplat_msg]` 在透明模式下自动为消息类型生成：
    - `TypeBinding<CppLang>`
    - `TypeBinding<PyLang>`

4. 不透明数据自动绑定

- `#[roplat::roplat_msg(lang = "cpp"|"py")]` 视为不透明结构：
    - 仅为对应语言生成 `TypeBinding` 实现
    - `is_opaque() == true`

5. 傀儡节点编译期约束

- `#[roplat::node(lang = "cpp"|"py", ...)]` 在宏展开时对以下类型做编译期检查：
    - 所有 input/output 的消息类型
    - 傀儡结构体字段类型
- 若类型未实现 `TypeBinding<目标语言>`，则编译失败并在节点定义处报错。
