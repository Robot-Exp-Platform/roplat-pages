# 节点

节点是 roplat 中最小的逻辑执行单元。

## 节点特征

Rust 原生节点实现 `Node` trait：

```rust
#[async_trait::async_trait]
pub trait Node: Send + Sync {
    type Input;
    type Output;
    type Error: Debug + Send;

    async fn process(&mut self, input: Self::Input) -> Result<Self::Output, Self::Error>;
    async fn on_init(&mut self) -> Result<(), Self::Error>;
    async fn on_shutdown(&mut self) -> Result<(), Self::Error>;
}
```

通常 `on_init` / `on_shutdown` 使用默认实现即可，`process` 是节点核心逻辑。

## 节点宏模式

`#[roplat::node]` 现在支持两种模式：

1. 原生节点模式（无参数）

- 用于 Rust 原生节点，继续沿用 `resource/param/state` 字段语义。

2. 多语言傀儡模式（带参数）

- 形式：`#[roplat::node(lang = "cpp"|"py", input(...), output(...))]`
- 用于声明 C++/Python 节点在 Rust 侧的“调度镜像”。
- 会在两遍扫描的 EXTRACT 阶段生成节点清单，供 `build.rs` 生成跨语言基类与用户实现模板。

## 编译期约束（多语言模式）

当使用 `#[roplat::node(lang = ...)]` 时，宏展开会对类型进行静态检查：

- 所有 input/output 消息类型必须实现 `TypeBinding<目标语言>`
- 傀儡结构体字段类型必须实现 `TypeBinding<目标语言>`

如果类型未绑定，编译会直接在节点定义处失败，而不是拖到运行时。

## 命名约定

为保持 Rust 调度名、生成文件名、跨语言类名一致，建议节点名直接使用目标类名（如 `CppController`、`PyPlanner`），不再额外附加后缀。
