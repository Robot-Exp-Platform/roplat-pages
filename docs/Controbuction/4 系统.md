# 系统

系统（System）是节点、节律源与通讯拓扑的封装单元。

## 系统职责

1. 组织节点执行顺序

- 将多个节点按拓扑关系组织成可调度图。

2. 管理节律驱动

- 由节律源触发节点运行。

3. 管理通讯通道

- 主流通讯：节点输入输出链路
- 旁路通讯：三缓冲等共享状态通道

4. 管理多语言节点接入

- Rust 侧使用 `#[roplat::node(lang = ...)]` 定义傀儡节点
- 构建阶段自动生成 C++/Python 基类与用户模板

## 编译期与构建期分工

系统能力分为两层：

1. 编译期（过程宏）

- 收集节点端口、字段、消息类型信息
- 在 EXTRACT 模式下写入 manifest

2. 构建期（build.rs）

- 读取 manifest
- 生成跨语言文件
- 可选编译 C++ 侧目标并链接

## 两遍扫描与系统装配

系统的多语言装配通过两遍扫描实现：

1. Pass 1（Extraction）

- `cargo check` 触发宏展开并产出 manifest

2. Pass 2（Generation）

- 读取 manifest 生成节点/消息文件

3. Final

- Rust 正常编译与链接，系统图与跨语言符号对齐

## 当前默认目录

- 系统管理文件：`cpp/roplat_gen`、`py/roplat_gen`
- 用户实现文件：`cpp/src`、`py`

其中系统文件可在 `build.rs` 中由 `BuildOrchestrator` 重新指定目录。
