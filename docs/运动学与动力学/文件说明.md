# 操作说明

## 文件说明

### 数据初步处理文件

+ 数据导入（第一步）
  + 使用`matlab`携带的数据导入功能，将你带来的数据导入到`matlab`中，并分别命名为`rho`和`lth`，对`rho`和`lth`分别右键保存为`.mat`文件格式
    + **`rho.mat`给出每个杆质量分布rho(n,100)，每一行为一个杆100等分每一份的质量，n行代表第n个杆**
    + **`lth.mat`给出每个杆长度lth(n,1)，每一行为一个杆的长度，n行代表第n个杆**
  + 也可以使用`rholthtest.m`文件来生成随机或满足一定规律的`rho`和`lth`数据，会自动将他们保存为`.mat`，如有需要请自行修改文件参数，注意矩阵维度的正确性。
+ 数据处理`data_process.m`（第二步）
  + 负责接收质量分布文件`rho.mat`长度文件`lth.mat`
  + 检查rho lth的正确性，并生成，最终程序运行所需要的数据并加载到工作区：`data_in`
  + 此外，需要提供初始位置`post/velc`矩阵
    + **post:位置矩阵，第一列为x，第二列为y，顺序从N到0，一共n+1行(第0位置请默认为0)**
    + **velc:速度矩阵，第一列为x，第二列为y，顺序从N到0，一共n+1行(第0位置请默认为0)**
+ 准备M矩阵（第三步）
  + M矩阵一般有两种设置模式，且维数必定与杆数量相等，注意运行时间匹配。
    1. 给定$M$矩阵，即整个系统在已知M的情况下进行系统运动求解，此时，M矩阵为1个时间列+n列M数据列，你可以使用`Mtestinit.m`文件来生成随机或满足一定规律的M数据，如有需要请自行修改文件参数，注意矩阵维度的正确性。
    2. 不给定$M$，但是每一时刻的M是由其他的数据计算获得，你可以在`simulink`中自行添加`Matlab Function`模块，并在模块中自定义$M$由$XY$或其他数据等计算而来，使用时请注意接线，以及每一个输出的维数匹配。注意使用此模式时，如果遇到报错，请①注意将函数输出的M维数匹配正确为从n-1到0的n*1形式矩阵，②请关注函数输出设置是否为离散时间步长(步长为$0.001s$)，③输出是否需要限制维度(使用`selector`模块选择$n\times 1$维度)

### 主程序

+ 主程序为`plant.slx`文件，请注意，不要随意更改模块内部连接。
+ 当前期工作准备好后，电机`simulink`中的更新模型，检查是否出现问题。
+ 模型计算：通过每一时刻$P,V,M$数据，计算得出$A,N$，然后通过积分计算下一刻的$P,V$,并且输出当前刻的每个关节的角度和角度变化量。
  + 运行过程中你可能会使用到$X,Y,N,V,A,Q$等数据，他们的输出形式为：
    + $X$:每个关节的x坐标，n+1行，每一行代表一个关节的x坐标，顺序为从N到0.
    + $Y$:每个关节的y坐标，n+1行，每一行代表一个关节的y坐标，顺序为从N到0.
    + $N$:每个关节的角度，2n行，前n行代表一个关节的X向作用力，后n行代表一个关节的Y向作用力，顺序为从N-1到0.
    + $V$:每个关节的角速度，n+1行2列，每一行代表一个关节的XY速度，顺序为从N到0.
    + $A$:每个关节的角加速度，n行2列，每一行代表一个关节的XY加速度，顺序为从N到1.(首端固定点加速度一定为0)
    + $Q$:每个关节的角度变化量，n行，每一行代表一个关节的角度变化量，顺序为从N-1到0.

### 数据展示

主程序运行中，你可以打开在`plant.slx`中的`XY Graph`用于查看每个关节的位置变化(需要自己手动选择一下，$XY$每一个数据一一对应即可)

+ 此外也可以通过提供的`plotxy.m`文件来进行数据展示
