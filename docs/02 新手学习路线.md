# 新手学习路线

本文面向完全不了解 Roplat 的新人。目标是让你在最短路径内，建立正确心智模型并能跑通一个最小示例。

本文采用“空白目录起步 + 依赖已发布库”的假设：你不需要先克隆 roplat 源码，直接通过包管理器安装依赖即可开始。

## 你会学到什么

1. Roplat 的核心理念是什么，解决了什么问题。
2. 节点、节律源、系统这三个核心抽象如何协作。
3. 如何理解主流通讯与旁路通讯。
4. 如何使用 two-pass 机制与多语言生成。
5. 如何在 `cc` 与 `cmake` 两种编译链中做选择。

## 建议实践节奏

1. 空目录完成基础环境检查

```powershell
rustc --version
cargo --version
git --version
```

1. 从零创建 Rust 基础工程并跑通

```powershell
cargo run
cargo test
```

1. 再升级到多语言拓展（默认 `cc` 后端）

```powershell
cargo build
cargo run
```

1. 再切换 `cmake` 后端验证

```powershell
$env:ROPLAT_NATIVE_BACKEND='cmake'
cargo build
```

1. 每次改动后执行质量门禁

```powershell
cargo clippy --all-targets -- -D warnings
```

## 学习中的常见误区

1. 误把“编译期约束”理解成“完全静态系统”

Roplat 追求的是“进程内静态 + 进程级动态”，并不是完全禁止动态行为。

1. 误把不透明类型当作 Rust 内部实体

当前设计下，不透明类型在 Rust 侧默认只做类型标记，具体字段和方法由目标语言用户文件实现。

1. 误把多语言看成“纯运行时插件”

Roplat 的多语言链路依赖构建期生成与编译，不只是运行时动态加载。
