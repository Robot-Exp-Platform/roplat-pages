# 概念导读

本部分回答一个问题：在真正写代码前，你需要先理解什么，才能少走弯路。

## 两层学习目标

1. 先理解概念边界：节点、消息、节律、系统、通讯。
2. 再理解工程边界：哪些文件是你写的，哪些文件是生成的。

## 先记住这三个原则

1. Rust 节点是基础能力。
2. C++/Python 节点是扩展能力。
3. 先跑通 Rust，再接入多语言。

## 文档分工

1. 概念讲解：解释“为什么这样设计”。
2. 快速开始：解释“如何从空目录一步步搭建完整项目”。
