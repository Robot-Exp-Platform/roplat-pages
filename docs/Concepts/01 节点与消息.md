# 节点与消息

## 节点是什么

节点是最小计算单元。它做两件事：

1. 接收输入
2. 产出输出

在工程实践中，建议一个节点只做一件核心业务逻辑。

## 消息是什么

消息是节点之间交换的数据结构。

建议：

1. 字段命名直观。
2. 避免一次塞太多无关字段。
3. 先保证能跑，再做字段优化。

## Rust 基础范式

典型工程里会有两个 Rust 文件：

1. `msg.rs`：定义消息
2. `nodes.rs`：定义节点

示例：

```rust
#[roplat::roplat_msg]
#[repr(C)]
#[derive(Clone, Debug, PartialEq)]
pub struct SensorData {
    pub x: f64,
    pub y: f64,
    pub z: f64,
    pub timestamp: u64,
}

pub struct MotorControllerNode {
    kp: f64,
}
```

## 新手常见误区

1. 一开始就拆太多节点，导致调试困难。
2. 消息字段设计过度，导致改动成本过高。
3. 还没跑通 Rust 就先接 C++/Python。
